# 🧪 ToxPredict: ML-Driven Drug Toxicity Screening

**ToxPredict** is an end-to-end Machine Learning pipeline and web application designed to predict the toxicity of chemical compounds at the early stages of drug discovery. By analyzing a compound's molecular structure, the system predicts the likelihood of toxicity and explicitly explains *why* using advanced SHAP (SHapley Additive exPlanations) visual analytics.

Built for the **Drug Toxicity Prediction Hackathon**.

---

## 1. Problem Statement
Drug development is a billion-dollar process that frequently fails in late stages due to unexpected compound toxicity. Identifying toxic properties computationally *before* physical synthesis or animal testing can drastically reduce R&D costs, accelerate discovery, and improve patient safety. Our goal is to predict chemical toxicity strictly from structural notations (SMILES).

## 2. Dataset
This project uses two complementary sources:
- **Tox21 (primary labels):** ~7,800 validated compounds. A molecule is labelled `1` if it is positive in **any** tested Tox21 assay, otherwise `0`.
- **ChEMBL withdrawn drugs (auxiliary labels):** a conservative set of withdrawn compounds added as extra toxic examples. These rows are kept **down-weighted** during training because they are useful but noisier than Tox21.

Current processed dataset snapshot:
- **7,853 compounds total**
- **7,823 Tox21 rows**
- **30 ChEMBL auxiliary rows**

## 3. The Pipeline
1. **Data Cleaning (`src/data_loader.py`)**: Validates raw SMILES strings using RDKit, removes unparseable molecules, canonicalizes structures, deduplicates them chemically, aggregates all Tox21 assays into one binary label, and merges the auxiliary ChEMBL supplement.
2. **Feature Engineering (`src/feature_engineering.py`)**: Converts SMILES strings into a dense 1,241-dimensional feature matrix:
   - *217 Molecular Descriptors* (Weight, LogP, TPSA, etc.)
   - *1024-bit Morgan Fingerprints* (ECFP4 standard to capture structural sub-graphs).
3. **Baseline Models (`src/baseline_models.py`)**: Trains Logistic Regression and Random Forest on the same stratified `70/15/15` split and the same continuous-only scaling contract used everywhere else.
4. **Model Tuning (`src/improve_model.py`)**: Trains the production **XGBoost Classifier** with simple precision-first tuning. ChEMBL rows are down-weighted by default (`0.5`) so Tox21 remains dominant.
5. **Ablation (`src/compare_chembl_experiment.py`)**: Compares XGBoost performance with vs. without ChEMBL and promotes the better run to `models/best_model.pkl`.
6. **Explainability (`src/shap_explain.py`, `src/explainability.py`)**: Uses SHAP on the selected production artifact and the exact same feature scaling contract used at inference time.
7. **Priority Toxin Bypass (`data/priority_toxins.json`)**: The Streamlit app checks a canonicalized toxin dictionary (76 high-priority toxins) before ML inference. Exact matches return **CRITICAL HAZARD** and bypass the model entirely.
8. **Precision-First Verdicts**: Non-dictionary predictions are triaged into **SAFE**, **UNCERTAIN**, or **CRITICAL HAZARD** using conservative thresholds so the app avoids overconfident binary calls.

## 5. Installation & Setup

Ensure you have **Python 3.9+** installed. We highly recommend using a virtual environment.

```bash
# 1. Clone the repository
git clone https://github.com/GoalSeekerHarsh/drug-toxicity-predictor.git
cd drug-toxicity-predictor

# 2. Create and activate a virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# 3. Install required dependencies
pip install -r requirements.txt
```

*(Note for macOS users: XGBoost requires OpenMP. If you encounter installation errors, run `brew install libomp` first).*

## 6. How to Run

### Option A — Quick Start with the Pre-Trained Model (Recommended)

A pre-trained XGBoost model (`models/tuned_xgboost_model.pkl`) is already included in this repository. You do **not** need to download data or retrain to use the app.

```bash
# Just launch the Streamlit app directly
streamlit run app/streamlit_app.py
```

Navigate to `http://localhost:8501`. Enter any valid SMILES string (e.g., `CCO` for Ethanol) to see real-time toxicity screening and SHAP breakdown plots. You can also upload a CSV file of SMILES strings in the **Batch Upload** tab.

> **Note:** The app loads `models/best_model.pkl` when present (generated by the ablation workflow), and falls back to `models/tuned_xgboost_model.pkl` automatically.

### Option B — Full Training Workflow (Retrain from Scratch)

If you want to retrain the model on fresh data, follow these steps in order:

**Prerequisites:** Download the [Tox21 dataset](https://tripod.nih.gov/tox21/challenge/) and place `tox21.csv` in `data/raw/`.

```bash
# 1. Clean and prepare the dataset
python -m src.data_loader

# 2. Compute molecular features (1,241 per compound)
python -m src.feature_engineering

# 3. Train baseline models (Logistic Regression, Random Forest)
python -m src.baseline_models

# 4. Train & tune the XGBoost model
python -m src.improve_model

# 5. Run ChEMBL ablation study and promote the best model
python -m src.compare_chembl_experiment

# 6. Generate SHAP explainability plots
python -m src.shap_explain

# 7. (Optional) Run ZINC virtual screening on 1,000 drug-like molecules
python -m src.zinc_screen

# 8. (Optional) Generate the final HTML/PDF report
python -m src.final_report

# 9. Launch the web app
streamlit run app/streamlit_app.py
```

All generated artifacts are saved automatically:
- Trained models → `models/`
- Evaluation metrics → `reports/*.json`
- Plots (ROC curves, SHAP, feature importance) → `reports/*.png`
- ZINC screening results → `reports/zinc_screen_results.csv`

## 7. Pre-Trained Model & Training History

### What Has Been Trained

The repository ships with a **pre-trained production model**:

| File | Description |
|------|-------------|
| `models/tuned_xgboost_model.pkl` | XGBoost trained on Tox21 (7,823 rows) + ChEMBL (30 rows), tuned with GridSearchCV for precision-first objective |

The pickle file stores a dictionary with four keys:
- `model` — the fitted `XGBClassifier`
- `scaler` — `StandardScaler` fitted on the training split (no data leakage)
- `feature_names` — ordered list of 1,241 feature names
- `thresholds` — conservative hazard/safe thresholds used by the app

### Modifications Made During Development

Key changes made after the initial prototype:
- **Added ChEMBL auxiliary data** (30 withdrawn drugs) with `sample_weight=0.5` down-weighting so Tox21 remains dominant.
- **Ablation study** (`src/compare_chembl_experiment.py`): automatically compares with/without ChEMBL and writes the winner to `models/best_model.pkl`.
- **Priority Toxin Dictionary** (`data/priority_toxins.json`): 76 high-confidence toxins (from ChEMBL + OSHA/EPA sources) bypass the ML model for deterministic `CRITICAL HAZARD` flagging.
- **Tri-state verdict** instead of binary: `SAFE` / `UNCERTAIN` / `CRITICAL HAZARD` avoids overconfident predictions near the decision boundary.
- **ZINC virtual screening** (`src/zinc_screen.py`): screens 1,000 ZINC-250k molecules; results stored in `reports/zinc_screen_results.csv`.

## 8. Model Results
Current ChEMBL ablation results on the held-out test split:

| Run | ROC-AUC | PR-AUC | Precision |
|-----|---------|--------|-----------|
| With ChEMBL (promoted) | 0.7731 | 0.7119 | **0.9487** |
| Without ChEMBL | 0.7804 | 0.6979 | 0.9057 |

Interpretation:
- The ChEMBL supplement remains an auxiliary signal and stays down-weighted.
- Promotion uses the same hazard threshold as the production app, so the reported precision matches live `CRITICAL HAZARD` behavior.
- On the current rebuilt dataset snapshot, the weighted ChEMBL run is the promoted production model because it delivers the highest held-out hazard precision.
- Precision is the primary promotion metric, with PR-AUC as the tie-breaker.
- The production app prefers `models/best_model.pkl` (written by the ablation step), and falls back to `tuned_xgboost_model.pkl`.

## 9. Feature Importance Summary
Using SHAP analysis, the top structural drivers indicating toxicity are:
1. **MolLogP** — lipophilicity (ability to cross membranes)
2. **BertzCT** — molecular complexity
3. **BCUT2D_LOGPHI** — LogP-weighted eigenvalue descriptor
4. **HeavyAtomMolWt** — molecular weight excluding hydrogen
5. **FractionCSP3** — fraction of sp3-hybridised carbons (3D shape proxy)
6. Morgan fingerprint bits capturing specific toxic substructures

The most current top-feature list is written to `reports/top_features.csv` after running `python -m src.shap_explain`.

## 10. Troubleshooting

| Problem | Likely Cause | Fix |
|---------|-------------|-----|
| `ModuleNotFoundError: rdkit` | RDKit not installed | `pip install rdkit` (see [RDKit docs](https://www.rdkit.org/docs/Install.html) if pip install fails on your platform) |
| `FileNotFoundError: models/best_model.pkl` | Ablation step not run | Run `python -m src.compare_chembl_experiment`, **or** the app will auto-fall back to `tuned_xgboost_model.pkl` |
| `FileNotFoundError: data/raw/tox21.csv` | Raw data missing | Download from [NIH Tox21](https://tripod.nih.gov/tox21/challenge/) and place in `data/raw/` |
| `FileNotFoundError: data/processed/features.npy` | Feature engineering not run | Run steps 1–2 in the full workflow (Option B above) |
| `streamlit: command not found` | Streamlit not on PATH | Use `python -m streamlit run app/streamlit_app.py` |
| App shows "Invalid SMILES" for valid molecule | RDKit canonicalization edge case | Try the canonical SMILES from [PubChem](https://pubchem.ncbi.nlm.nih.gov/) |
| macOS XGBoost install error | OpenMP missing | `brew install libomp`, then `pip install xgboost` |
| `KeyError` in SHAP plot | Feature mismatch after retrain | Delete `models/` and rerun the full training workflow |

## 11. Future Work
If given more time past this hackathon, we would explore:
- **Graph Neural Networks (GNNs):** Implementing Chemprop or DGL to learn directly from the molecular graph structure rather than relying entirely on pre-computed RDKit descriptors.
- **Multi-task Learning:** Predicting the 12 Tox21 endpoints as separate but related tasks instead of using the current aggregated "any-assay-positive" label.
- **Cloud Deployment:** Containerizing the Streamlit application using Docker and deploying it to AWS Fargate or Google Cloud Run for public access.
